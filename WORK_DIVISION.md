# Project Division & Implementation Plan

This project naturally splits into two main areas: **Data Engineering/Processing** and **Model Architecture/Training**. This division allows both engineers to work independently on their local/cloud systems, integrating via a shared data format (e.g., Parquet or TFRecords stored in HDFS).

Here is an equal division of the implementation workload.

---

## Role 1: Big Data & Spark Engineer (Hadoop/PySpark)
**Focus:** Distributed data processing, pipeline building, and large-scale data science operations on a cluster or local PySpark environment.

### Responsibilities
1. **Distributed Data Ingestion & Storage:**
   - Migrate raw FER2013 CSV files and any new custom datasets (e.g., LFW) into the Hadoop Distributed File System (HDFS).
   - Write scripts to efficiently parse raw datasets using PySpark DataFrames.

2. **Distributed Preprocessing (`spark_preprocess.py`):**
   - Implement the data science logic required before neural networks touch the data.
   - Use PySpark User-Defined Functions (UDFs) to process image string strings into numpy arrays, resize (e.g., 64x64 or 48x48), and normalize pixel values [0-1].
   - **Synthetic Mask Generation:** Implement the logic that overlays synthetic masks on the raw images using OpenCV inside a PySpark job, generating the `Masked -> Original` pairs required for the autoencoder.

3. **Data Augmentation:**
   - Perform distributed augmentations (random flips, rotations, contrasting) to expand the dataset in PySpark, reducing the load on the Deep Learning training pipeline.

4. **Optimized Output:**
   - Save the finalized, fully processed data into a distributed format like Parquet or TFRecords back to HDFS, separated cleanly into `Train`, `Validation`, and `Test` sets. 

---

## Role 2: Deep Learning & MLOps Engineer (TensorFlow/Keras)
**Focus:** Building, tuning, training, and deploying neural network architectures.

### Responsibilities
1. **Model Architecture Design (`models.py`):**
   - Refine the Unmasking Autoencoder architecture, ensuring it effectively reconstructs lower facial features.
   - Implement and tune the ResNet or similar advanced convolutional architecture for the emotion classification task on the reconstructed faces.

2. **Data Pipeline Integration (`data_loader.py`):**
   - Modify the deep learning pipeline to read efficiently from the Parquet/TFRecord files generated by Role 1. Use the `tf.data` API to handle prefetching and batching directly from HDFS or local disk.

3. **Model Training & Callbacks (`train.py`):**
   - Construct the training loops for both the Autoencoder and the Classifier.
   - Implement Keras Callbacks (Early Stopping, TensorBoard logging, Checkpointing) to monitor and optimize training over multiple epochs.
   - Save the final optimized `.h5` or SavedModel formats (`emotion_model.h5`, `unmasking_model.h5`).

4. **Evaluation & Inference (`evaluate_results.py`, `predict.py`):**
   - Develop evaluation scripts that take the test set, run it through the masking -> unmasking -> detection pipeline, and generate Confusion Matrices and classification metrics.
   - Create prediction scripts capable of running inference on new standalone images.

---

## Integration Plan (The "Handshake")
The central integration point between both roles is the **Processed Dataset**.

- **Agreed Format:** Both roles must agree on the format of the output from the Spark job (e.g., Parquet).
- **Agreed Schema:** For example, columns might be: `[masked_pixels_array, unmasked_pixels_array, emotion_label_int, usage_string]`.
- Once Role 1 pushes a successful Spark job output to HDFS/Disk, Role 2 can simply point their `tf.data` loader to that directory and begin experiments without overlapping code files.
